# 3.3.1.blueprint.product.v1

## summary

implement tool use support for rhachet-brains-chutes brain atoms, mirrored from the rhachet-brains-xai implementation. the implementation adds:

1. **cast functions** — convert between rhachet and openai tool formats
2. **ask signature extension** — add TPlugs generic and tool-aware prompt type
3. **tool call extraction** — detect and convert tool_calls from api response
4. **tool continuation** — support prompt as tool execution results

the pattern follows openai-compatible function call api, which chutes supports via vllm backend.

## contracts

### rhachet types (from peer dependency)

| type | purpose |
|------|---------|
| `BrainPlugToolDefinition` | tool schema with slug, name, description, zod schemas |
| `BrainPlugToolInvocation` | output when brain requests tool execution |
| `BrainPlugToolExecution` | input when developer feeds tool results back |
| `BrainOutput<TOutput, 'atom', TPlugs>` | response type with `calls.tools` field |
| `BrainAtomPlugs` | plugs type with optional `tools` array |

### openai types (from openai sdk)

| type | purpose |
|------|---------|
| `ChatCompletionTool` | tool definition for api request |
| `ChatCompletionMessageToolCall` | tool call from api response |
| `ChatCompletionMessageParam` | message for api request (includes tool role) |

## domain decomposition

```
brain atom (chutes)
  ├── tool definition cast
  │     └── castIntoChutesToolDef: BrainPlugToolDefinition → ChatCompletionTool
  ├── tool call extraction
  │     └── castFromChutesToolCall: ChatCompletionMessageToolCall → BrainPlugToolInvocation
  ├── tool continuation cast
  │     └── castIntoChutesToolMessages: BrainPlugToolExecution[] → ChatCompletionMessageParam[]
  └── ask orchestration
        ├── prompt type dispatch (string vs tool executions)
        ├── tools parameter injection
        ├── tool_calls extraction
        └── conditional output (null when tools called)
```

## codepath maintainability

### separation of concerns

| layer | responsibility |
|-------|----------------|
| cast functions | pure conversion between formats |
| genBrainAtom | orchestration and api interaction |
| tests | behavioral verification |

### single responsibility

each cast function has one job:
- `castIntoChutesToolDef` — converts tool definition
- `castFromChutesToolCall` — converts tool call
- `castIntoChutesToolMessages` — converts tool executions to messages

### readability

- cast functions are small (< 20 lines each)
- genBrainAtom changes are additive, not rewrite
- tool use branch is clearly guarded by `if (plugs?.tools?.length)`

## edge cases

| edgecase | behavior |
|----------|----------|
| no tools plugged | behaves as before (no tools parameter) |
| tools plugged, brain answers directly | output populated, calls null |
| tools plugged, brain calls tool | output null, calls populated |
| tool execution error | error signal fed back, brain adapts |
| empty tool_calls array | treated as no tools called |
| malformed tool arguments | JSON.parse may throw, propagate error |
