### issue 1: edge 1 prompt content mismatch

**issue detected**: playtest edge 1 uses prompt "what is the capital of france?" with expected output "Paris", but integration test [case2] uses "say hello" with expected output "hello".

**impact analysis**:
- behavioral contract is identical: no tools plugged → direct output, calls is null
- only the specific content differs
- both prompts verify the same code path

**verdict**: acceptable variance. the behavioral contract is verified. the playtest could use either prompt — both prove the same behavior.

**disposition**: documented as acceptable. no fix needed because:
1. code path coverage is identical
2. playtest documents behavior, not specific words
3. [case2] predates tool use feature (it tests basic ask functionality)
