### why usecase.3 test was omitted

the chutes api (like openai's) has mutual exclusivity between `response_format: { type: 'json_schema' }` and `tools`. when tools are plugged:
- the api call includes `tools` parameter
- the api call cannot also include `response_format` with json_schema
- therefore, the model may return unstructured text, not JSON

this means we cannot reliably test "brain answers directly with tools plugged" because the direct answer would not be JSON-parseable. the code handles this (lines 128-142), but the test would be flaky.
