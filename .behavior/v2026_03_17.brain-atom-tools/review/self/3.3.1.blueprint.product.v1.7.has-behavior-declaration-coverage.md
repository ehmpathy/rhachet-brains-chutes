## gaps evaluated

### gap.1: model compatibility documentation

the criteria mention "model does not support tools" as undefined behavior.

**why acceptable**: this is intentionally undefined. model compatibility is consumer's responsibility. a model capability registry would require maintenance and is YAGNI (documented in `3.3.1.blueprint.product.v1.3.has-pruned-yagni.md`).

### gap.2: case7 removal

usecase.3 (direct answer with tools) was blueprinted as case7 but removed.

**why acceptable**: the chutes api has mutual exclusivity between `response_format` and `tools`. without `response_format`, model may return plain text instead of json. the behavior exists in code (returns null calls if no tool_calls) but cannot be reliably tested. documented in prior reviews.
