### question 3: does chutes return `tool_calls` in same format as openai?

**triage: [research] — but low risk**

**what I tried first:**

I reasoned about the SDK type enforcement:
- chutes uses `OpenAI` sdk with `baseURL: 'https://llm.chutes.ai/v1/'`
- the response is typed as `OpenAI.ChatCompletion`
- `response.choices[0].message.tool_calls` has type `OpenAI.ChatCompletionMessageToolCall[]`
- TypeScript enforces this structure at compile time

**why this suggests the answer is likely [yes]:**
- the openai sdk validates response shapes
- if chutes returned a different structure, the sdk would error
- xai uses the same pattern and it works

**why I cannot confidently answer [yes] now:**
- sdk types are compile-time; runtime could return unexpected shapes
- chutes could return extra fields or omit optional fields
- edge cases (empty tool_calls, malformed arguments) are unknown
- no integration test has verified this yet

**why it holds as [research]:**
- verifiable in first integration test (< 5 minutes)
- low risk: sdk types provide strong confidence
- castFromChutesToolCall will adapt if minor differences exist
