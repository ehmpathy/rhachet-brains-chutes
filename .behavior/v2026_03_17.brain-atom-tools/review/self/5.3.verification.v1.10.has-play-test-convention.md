## why no .play.test.ts

this repo uses a different convention:
- **unit tests**: `*.unit.test.ts` (via jest.unit.config.ts)
- **integration tests**: `*.integration.test.ts` (via jest.integration.config.ts)
- **acceptance tests**: `*.acceptance.test.ts` (via jest.acceptance.config.ts)

journey tests are embedded in integration tests:
- genBrainAtom.integration.test.ts contains journey cases (case5, case6, case8, case9)
- each case follows given/when/then structure
