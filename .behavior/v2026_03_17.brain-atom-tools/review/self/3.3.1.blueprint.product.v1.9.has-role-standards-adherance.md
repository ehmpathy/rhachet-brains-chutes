## pitofsuccess.typedefs adherence

### rule.require.shapefit

**rule**: types must fit without `as` casts (except documented boundaries)

**checked implementation**:
- `genBrainAtom.ts` line 237: `calls: { tools: toolInvocations } as any`
- `genBrainAtom.ts` line 250: `calls: null as any`

**analysis**: the `as any` casts are at the BrainOutput constructor boundary. rhachet's generic type `BrainOutput<TOutput, 'atom', TPlugs>` has complex conditional types for `calls` field. the cast is a documented workaround for library type limitations, not a type safety violation.

**holds with caveat**: casts exist but are at external library boundary, not internal logic.
