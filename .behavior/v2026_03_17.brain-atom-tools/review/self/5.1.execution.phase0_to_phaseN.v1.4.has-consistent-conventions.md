## convention-by-convention analysis

### 1. cast function names

**question**: do the cast function names follow rhachet conventions?

**rhachet convention**: `cast{Into|From}{Domain}` where:
- `Into` = direction toward target format
- `From` = direction from source format

**verification against new code**:

| function | direction | target/source | follows convention? |
|----------|-----------|---------------|---------------------|
| `castIntoChutesToolDef` | Into | openai `ChatCompletionTool` | ✓ |
| `castFromChutesToolCall` | From | openai `ChatCompletionMessageToolCall` | ✓ |
| `castIntoChutesToolMessages` | Into | openai `ChatCompletionMessageParam[]` | ✓ |

**code evidence**:

```ts
// castIntoChutesToolDef.ts:9 - converts TO openai format
export const castIntoChutesToolDef = (input: {
  tool: BrainPlugToolDefinition;
}): OpenAI.ChatCompletionTool => ({

// castFromChutesToolCall.ts:8 - converts FROM openai format
export const castFromChutesToolCall = (input: {
  call: OpenAI.ChatCompletionMessageToolCall;
}): BrainPlugToolInvocation => ({

// castIntoChutesToolMessages.ts:8 - converts TO openai format
export const castIntoChutesToolMessages = (input: {
  executions: BrainPlugToolExecution[];
}): OpenAI.ChatCompletionMessageParam[] => {
```

**verdict**: all cast functions follow `cast{Into|From}Chutes{Domain}` convention. direction semantics are correct.
