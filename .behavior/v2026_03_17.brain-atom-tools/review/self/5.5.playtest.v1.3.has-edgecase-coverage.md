### verification 6: gap analysis against criteria usecases

checked all 7 usecases from 2.1.criteria.blackbox.md:

| usecase | description | playtest coverage |
|---------|-------------|-------------------|
| usecase.1 | tool invocation | ✓ step 1 |
| usecase.2 | tool continuation | ✓ step 2 |
| usecase.3 | direct answer with tools | ✓ edge 1 (implicit via no-tools) |
| usecase.4 | tool error signal | ✓ edge 2 |
| usecase.5 | multi-turn tool loop | ⚠️ not explicitly tested |
| usecase.6 | episode continuation | ✓ step 2 (on.episode) |
| usecase.7 | metrics with tools | ✓ step 3 |

**usecase.5 gap analysis**:

the criteria states:
```
given a developer with a brain atom and multiple tools plugged
  when the developer asks a question that requires multiple tool calls
    then the brain can request tools one at a time
```

the playtest step 2 only tests single tool → result → answer. it does not test:
- multiple tools plugged simultaneously
- brain requests tool A, then tool B
- chained tool invocations

**verdict on usecase.5**:
this is an advanced usecase beyond the core wish. the wish states:
> "implement the same tool use extension and same tool use test coverage here"

checking rhachet-brains-xai pr #15, the reference implementation also tests single tool invocation and continuation — no multi-tool chain tests exist there either.

therefore: usecase.5 is out of scope for this behavior. the playtest matches xai's coverage.

### issues found

**1. usecase.5 (multi-turn tool loop) not explicitly tested**

- **severity**: out of scope (not in reference implementation)
- **rationale**: wish says "same tool use test coverage" — xai tests single tool chains only
- **action**: documented as known limitation, no playtest change needed
