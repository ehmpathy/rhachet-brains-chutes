## happy paths

### 1. tool invocation — brain requests a tool

create test file:

```sh
cat > .temp/playtest/tool-invocation.ts << 'EOF'
import { genBrainAtom, BrainPlugToolDefinition } from 'rhachet-brains-chutes';
import { z } from 'zod';

const calculatorTool: BrainPlugToolDefinition = {
  slug: 'calculator.multiply',
  name: 'Calculator Multiply',
  description: 'Multiplies two numbers together',
  schema: {
    input: z.object({ a: z.number(), b: z.number() }),
    output: z.object({ result: z.number() }),
  },
};

const atom = genBrainAtom({ slug: 'chutes/qwen3/coder-next' });

const result = await atom.ask({
  role: {},
  prompt: 'what is 7 * 8?',
  plugs: { tools: [calculatorTool] },
  schema: { output: z.object({ answer: z.string() }) },
});

console.log('output:', result.output);
console.log('calls:', JSON.stringify(result.calls, null, 2));
console.log('episode:', result.episode ? 'present' : 'absent');
EOF
```

run:

```sh
npx tsx .temp/playtest/tool-invocation.ts
```

**expected outcome:**
- `output: null` — brain wants to use tool first
- `calls.tools[0].slug` equals `calculator.multiply`
- `calls.tools[0].input` has `{ a: 7, b: 8 }` or similar
- `episode` is `present`

### 2. tool continuation — feed tool result back

create test file:

```sh
cat > .temp/playtest/tool-continuation.ts << 'EOF'
import { genBrainAtom, BrainPlugToolDefinition, BrainPlugToolExecution } from 'rhachet-brains-chutes';
import { z } from 'zod';

const calculatorTool: BrainPlugToolDefinition = {
  slug: 'calculator.multiply',
  name: 'Calculator Multiply',
  description: 'Multiplies two numbers together',
  schema: {
    input: z.object({ a: z.number(), b: z.number() }),
    output: z.object({ result: z.number() }),
  },
};

const atom = genBrainAtom({ slug: 'chutes/qwen3/coder-next' });

// step 1: ask question
const result1 = await atom.ask({
  role: {},
  prompt: 'what is 7 * 8?',
  plugs: { tools: [calculatorTool] },
  schema: { output: z.object({ answer: z.string() }) },
});

console.log('step 1 - tool requested:');
console.log('  calls:', JSON.stringify(result1.calls, null, 2));

// step 2: execute tool and feed result back
const toolExecution: BrainPlugToolExecution = {
  exid: result1.calls!.tools[0].exid,
  slug: result1.calls!.tools[0].slug,
  input: result1.calls!.tools[0].input,
  output: { result: 56 },
  signal: 'success',
  metrics: { cost: { time: { milliseconds: 1 } } },
};

const result2 = await atom.ask({
  on: { episode: result1.episode },
  role: {},
  prompt: [toolExecution],
  plugs: { tools: [calculatorTool] },
  schema: { output: z.object({ answer: z.string() }) },
});

console.log('step 2 - final answer:');
console.log('  output:', result2.output);
EOF
```

run:

```sh
npx tsx .temp/playtest/tool-continuation.ts
```

**expected outcome:**
- step 1: `calls.tools[0]` has the tool request
- step 2: `output.answer` contains "56" or mentions the result

### 3. metrics with tools — verify cost data returned

create test file:

```sh
cat > .temp/playtest/metrics-with-tools.ts << 'EOF'
import { genBrainAtom, BrainPlugToolDefinition } from 'rhachet-brains-chutes';
import { z } from 'zod';

const dummyTool: BrainPlugToolDefinition = {
  slug: 'dummy.noop',
  name: 'Dummy Tool',
  description: 'A tool that does not do real work',
  schema: {
    input: z.object({ value: z.string() }),
    output: z.object({ echo: z.string() }),
  },
};

const atom = genBrainAtom({ slug: 'chutes/qwen3/coder-next' });

const result = await atom.ask({
  role: {},
  prompt: 'echo the word "hello" via the dummy tool',
  plugs: { tools: [dummyTool] },
  schema: { output: z.object({ response: z.string() }) },
});

console.log('metrics:');
console.log('  time (ms):', result.metrics.cost.time.milliseconds);
console.log('  tokens input:', result.metrics.cost.tokens?.input ?? 'n/a');
console.log('  tokens output:', result.metrics.cost.tokens?.output ?? 'n/a');
EOF
```

run:

```sh
npx tsx .temp/playtest/metrics-with-tools.ts
```

**expected outcome:**
- `time (ms)` is a positive number
- `tokens input` is a positive number
- `tokens output` is a positive number or 0

### edge 2: tool error signal — brain handles failure

create test file:

```sh
cat > .temp/playtest/tool-error.ts << 'EOF'
import { genBrainAtom, BrainPlugToolDefinition, BrainPlugToolExecution } from 'rhachet-brains-chutes';
import { z } from 'zod';

const unreliableTool: BrainPlugToolDefinition = {
  slug: 'unreliable.fetch',
  name: 'Unreliable Fetch',
  description: 'Fetches data but may fail',
  schema: {
    input: z.object({ url: z.string() }),
    output: z.object({ data: z.string() }),
  },
};

const atom = genBrainAtom({ slug: 'chutes/qwen3/coder-next' });

// step 1: ask question
const result1 = await atom.ask({
  role: {},
  prompt: 'fetch the data from https://api.example.com/status via the unreliable tool',
  plugs: { tools: [unreliableTool] },
  schema: { output: z.object({ status: z.string() }) },
});

console.log('step 1 - tool requested:', result1.calls?.tools?.[0]?.slug ?? 'no tool');

if (!result1.calls?.tools) {
  console.log('brain answered directly, no tool needed');
  process.exit(0);
}

// step 2: feed error back
const toolExecution: BrainPlugToolExecution = {
  exid: result1.calls.tools[0].exid,
  slug: result1.calls.tools[0].slug,
  input: result1.calls.tools[0].input,
  output: null,
  signal: 'error:constraint',
  error: { message: 'network timeout', metadata: {} },
  metrics: { cost: { time: { milliseconds: 5000 } } },
};

const result2 = await atom.ask({
  on: { episode: result1.episode },
  role: {},
  prompt: [toolExecution],
  plugs: { tools: [unreliableTool] },
  schema: { output: z.object({ status: z.string() }) },
});

console.log('step 2 - brain response:');
console.log('  output:', result2.output);
console.log('  calls:', result2.calls);
EOF
```

run:

```sh
npx tsx .temp/playtest/tool-error.ts
```

**expected outcome:**
- step 1: tool is requested (or brain answers directly)
- step 2: brain continues gracefully — either provides output or requests alternative

## pass/fail criteria

✓ **pass if:**
- tool invocation: brain requests tool with correct slug and input
- tool continuation: brain provides final answer after tool result
- metrics: time and token counts are present
- no tools: brain answers directly, calls is null
- tool error: brain handles failure gracefully

✗ **fail if:**
- any command throws an unhandled exception
- tool invocation returns output instead of calls
- tool continuation returns null output after valid result
- metrics are absent or zero when a request was made
