## 3. new test cases required

### 3.1 [case5] tool invocation [NEW]

**purpose**: verify brain requests tool when appropriate

**xai reference pattern**:
```ts
given('[case5] tool invocation', () => {
  const calculatorTool: BrainPlugToolDefinition = {
    slug: 'calculator.multiply',
    name: 'Calculator Multiply',
    description: 'multiplies two numbers together',
    schema: {
      input: z.object({
        a: z.number().describe('first number'),
        b: z.number().describe('second number'),
      }),
      output: z.object({ result: z.number() }),
    },
  };

  when('[t0] asked to multiply with tool available', () => {
    then('it should request the tool', async () => {
      const result = await atom.ask({
        role: {},
        prompt: 'use the calculator to multiply 7 by 8',
        plugs: { tools: [calculatorTool] },
        schema: { output: z.object({ answer: z.string() }) },
      });
      expect(result.output).toBeNull();
      expect(result.calls?.tools).toBeDefined();
      expect(result.calls?.tools?.length).toBeGreaterThan(0);
      expect(result.calls?.tools?.[0].slug).toEqual('calculator.multiply');
      expect(result.calls?.tools?.[0].input).toMatchObject({ a: 7, b: 8 });
    });
  });
});
```

given('[case6] tool continuation', () => {

          signal: 'success',
          output: { result: 56 },
          metrics: { cost: { time: { milliseconds: 1 } } },
        }],
        plugs: { tools: [calculatorTool] },
        schema: { output: z.object({ answer: z.string() }) },
      });
      expect(result2.output).toBeDefined();
      expect(result2.output?.answer).toContain('56');
    });
  });
});

## 6. test execution considerations

### 6.1 api rate limits

chutes may have rate limits. consider:
- add delay between tests if needed
- use `jest.setTimeout()` for longer tests
- handle "no instances available" errors gracefully

### 6.2 model variability

tool use behavior may vary by model. the tests should:
- use explicit tool descriptions ("always use this for...")
- allow some flexibility in assertion (e.g., "contains" vs "equals")
- document expected behavior vs observed behavior

### 6.3 test isolation

each test case should be independent:
- no shared state between cases
- fresh atom instance per describe block
- tool definitions defined within test scope
