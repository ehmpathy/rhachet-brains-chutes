# blueprint: rhachet-brains-chutes

> implementation plan to convert this repo from a together ai brain supplier adapter to a chutes brain supplier adapter

---

## 1. summary

replace all together ai references with chutes equivalents across the entire repo:

| dimension | together ai (before) | chutes (after) |
|-----------|---------------------|----------------|
| package name | `rhachet-brains-togetherai` | `rhachet-brains-chutes` |
| base url | `https://api.together.xyz/v1` | `https://llm.chutes.ai/v1/` |
| env var | `TOGETHER_API_KEY` | `CHUTES_API_KEY` |
| slug prefix | `together/` | `chutes/` |
| type name | `TogetherBrainAtomSlug` | `ChutesBrainAtomSlug` |
| sdk export | `getBrainAtomsByTogetherAI` | `getBrainAtomsByChutes` |
| model count | 10 | 11 (8 carried + 3 chutes-exclusive) |

the openai sdk pattern, `genBrainAtom` factory, `BrainAtom` contract, episode continuation, and structured output all remain identical вЂ” only the supplier-specific config changes.

## 8. risks and mitigations

| risk | impact | mitigation |
|------|--------|------------|
| chutes-exclusive model rates unverified | incorrect BrainSpec cost | verify via `/v1/models` endpoint or chutes dashboard before merge |
| 503 "no instances available" in integration tests | flaky test suite | use `then.repeatably({ attempts: 3, criteria: 'SOME' })` for multi-model tests |
| model id mismatch (chutes vs verified) | 404 errors on ask | verify each model id via live api call in integration tests |
| kimi-k2.5 is tee-only variant on chutes | different behavior than together ai | document in description and readme; rates confirmed ($0.60/$3.00) |

| chutes rate instability (tao-dependent) | rates may drift from documented values | note in readme that rates are approximate and tao-subsidized |
